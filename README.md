## Расположение репозитория
https://github.com/Gw1n/git_course.git
Для дублирования кода в локальный репозиторий использовать:
```git pull <name> <brunch>```

## Процесс внесения изменений в основную кодовую базу
Основных кодовых веток две:
1. master - релизная (стабильная) версия ПО
2. develop - дев (нестабильная) версия ПО

Все изменения: разработка фич, багфиксы и хотфиксы производятся в ответвлениях (о порядке наименований ниже), после чего создается pull-request для слияния с основной кодовой веткой. Слияние происходит только после одобрения тех. лидом.

## Правила наименования веток
1. ```feature/<name>``` - ответвление от ветки develop для разработки/улучшение фичи, содержит краткое описание фичи после ```/```, мержится только обратно в develop
2. ```hotfix/<name>``` - ответвление от ветки master для экстренного исправление неисправностей релизной версии, содержит номер тикета, первого зарепортившего баг, после ```/```, мержится только обратно в master
3. ```releale/<name>``` - ветка, содержащая версию ПО в пограничном состоянии с релизом, предназначена для отлавливания багов в предрелизной и дев версиях ПО, содержит краткое описание бага после ```/```, мержится в любую из основных веток в зависимости от состояния

## Настройка среды разработки
Рекомендуется использовать IDE от JetBrains.
1. Для подключения удаленного репозитория: ```Настройки -> Удаленный репозиторий -> GitHub```, затем войти в свой аккаунт через браузер
2. Слева сверху инициализировать локальный репозиторий по нажатию на соответствующую кнопку
3. Через терминал в IDE подключить удаленный репозиторий по ссылке (указана сверху), написав ```git remote add <name> <source>```
4. Также скачать себе конфиг https://github.com/Gw1n/git_course/blob/master/gitconfig
   Или применить необходимые настройки вручную прописав в терминале ```git config --global alias.history "log --graph"```

## Шпоргалка по работе с ветками
```git branch <name>``` - создание новой ветки
```git checkout <name>``` - переключение на новую ветку
1. ```git add <list of files>``` - добавить желаемые изменений в буфер
2. ```git commit -m "<message>"``` - закоммитить изменения в локальном репозитории с определенным описанием
3. ```git push <name> <branch>``` - пуш в удаленные репозиторий (указывать имя, присвоенное удаленному репозиторию при добавлении и название желаемой ветки)
```git merge <name>``` - слияние указанной ветки в текущую
